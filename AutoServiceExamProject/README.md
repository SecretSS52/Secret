# AutoServiceCalculator

## Назначение проекта

DLL-библиотека для расчета стоимости заказа в автосервисе. Проект сделан как пример для задания третьего уровня по МДК 02.02.

## Что реализовано

- Клиент автосервиса.
- Автомобиль клиента.
- Услуги автосервиса.
- Заказ клиента.
- Система скидок.
- Расчет итоговой стоимости.
- Пользовательское исключение.
- Debug/Trace логирование.
- Модульные тесты MSTest.
- CI/CD через GitHub Actions.

## Структура проекта

```text
AutoServiceLibrary — основной DLL-проект.
AutoServiceTests — проект модульного тестирования.
.github/workflows — настройки CI/CD.
```

## Основные классы

`Client` — клиент автосервиса.  
`Car` — автомобиль клиента.  
`ServiceItem` — одна услуга автосервиса.  
`ServiceOrder` — заказ клиента.  
`OrderCalculator` — расчет стоимости заказа.  
`TraceLogger` — настройка Trace-записи в файл.  
`OrderValidationException` — пользовательское исключение.

## Как работает расчет

1. Создается клиент.
2. Создается автомобиль.
3. Создается заказ.
4. В заказ добавляются услуги.
5. Суммируется стоимость услуг.
6. Определяется скидка клиента.
7. Рассчитывается итоговая сумма.

Формула:

```text
итог = сумма услуг - скидка
```

Пример:

```text
Диагностика = 1500
Замена масла = 2500
Сумма = 4000
Gold = скидка 10%
Скидка = 400
Итог = 3600
```

## Система скидок

```text
Regular — 0%
Silver  — 5%
Gold    — 10%
```

## Отладочная информация

Используются:

```csharp
Debug.WriteLine();
Trace.WriteLine();
```

`Debug.WriteLine()` выводит сообщения в окно Output / Вывод Visual Studio.  
`Trace.WriteLine()` записывает сообщения в файл `autoservice_log.txt`.

Файл лога создается в папке запуска, например:

```text
AutoServiceTests/bin/Debug/net8.0/autoservice_log.txt
```

## Модульное тестирование

Тестовый проект: `AutoServiceTests`.

Тесты проверяют:

- расчет без скидки;
- расчет со скидкой Silver;
- расчет со скидкой Gold;
- ошибку при заказе без услуг;
- ошибку при отрицательной цене услуги;
- ошибку при пустом имени клиента.

Запуск тестов:

```bash
dotnet test
```

## Что сказать на экзамене

Я реализовал DLL-проект `AutoServiceLibrary`, предназначенный для расчета стоимости заказа в автосервисе.

В проекте используется несколько классов: `Client`, `Car`, `ServiceItem`, `ServiceOrder`, `OrderCalculator`, `TraceLogger`.
Основной функционал находится в классе `OrderCalculator`, который суммирует стоимость услуг, определяет скидку клиента и возвращает итоговую сумму.

Для проверки корректности данных используются стандартные исключения и пользовательское исключение `OrderValidationException`.

Для диагностики используются `Debug.WriteLine()` и `Trace.WriteLine()`.
Отладочная информация выводится в окно отладки и записывается в текстовый файл.

Для проверки функционала создан проект модульных тестов `AutoServiceTests`.
Тесты проверяют расчет стоимости для разных уровней клиента и обработку ошибочных данных.

Проект добавлен в Git, загружен на GitHub, а CI/CD настроен через GitHub Actions.
При отправке изменений автоматически выполняются сборка проекта и запуск тестов.
