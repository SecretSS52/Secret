--1 --Создание репозитория git с использованием командной строки. Добавление файлов и фиксация изменений, создание веток их слияние.
git init

echo "# Task 1 project" > README.md
echo "Console.WriteLine(\"Hello Git\");" > Program.cs

git add .
git commit -m "Initial commit"

git checkout -b feature
echo "Feature code" > feature.txt
git add .
git commit -m "Add feature file"

git checkout master
git merge feature

--2 --По средствами использования командной строки git и команды git rebase, скопировать некоторые коммиты с одной ветки, на новую ветку.
Да, у тебя в целом получилось: ты через rebase --onto взял часть коммитов из second_branch и перенёс их в новую ветку third_branch поверх master.

Краткая инструкция:

git init

Создаём первый коммит в master:

git add .
git commit -m "added 2 empty files"

Создаём ветку second_branch:

git checkout -b second_branch

Делаем несколько коммитов в second_branch:

echo "" > file3
git add .
git commit -m "add file 3"

echo "" > file4.txt
git add .
git commit -m "add file 4"

echo "" > file5.txt
git add .
git commit -m "add file 5"

echo "change 1" >> file4.txt
git add .
git commit -m "change file 4"

echo "change 2" >> file5.txt
git add .
git commit -m "change file 5"

Смотрим историю и выбираем нужный коммит:

git log --oneline

Переходим на нужный коммит и создаём новую ветку:

git checkout 28b1e95
git branch third_branch
git checkout third_branch

Делаем rebase --onto:

git rebase --onto master 4c916eb third_branch

Расшифровка:

master — новая основа, куда переносим коммиты
4c916eb — старое основание, после него берём коммиты
third_branch — ветка, которую переносим

То есть команда значит:

Взять коммиты из third_branch после 4c916eb
и перенести их поверх master.

Если появляется конфликт:

git add .
git rebase --continue

Если Git попросит сообщение коммита, сохраняешь его и выходишь из редактора.

Проверяем результат:

git log --oneline --graph --all --decorate

Итог должен быть примерно такой:

новый коммит ← third_branch
новый коммит
| * старые коммиты ← second_branch
|/
первый коммит ← master

--3 --Использование git cherry-pick, скопировать коммиты с одной ветки на другую

git init

echo "Start" > file.txt
git add .
git commit -m "Initial commit"

git checkout -b source

echo "Useful commit 1" >> file.txt
git add .
git commit -m "Useful commit 1"

echo "Useful commit 2" >> file.txt
git add .
git commit -m "Useful commit 2"

git log --oneline

git checkout master
git checkout -b target

git cherry-pick HASH_КОММИТА_1

--6 --По средствами использования командной строки git построить полный граф истории. Скопировать один из файлов с предыдущих коммитов в текущий коммит.

git init

echo "Version 1" > file.txt
git add .
git commit -m "Version 1"

echo "Version 2" > file.txt
git add .
git commit -m "Version 2"

echo "Version 3" > file.txt
git add .
git commit -m "Version 3"

git log --oneline --graph --all

Допустим, нужен файл из первого коммита. Сначала смотри хэш:
git log --oneline

Потом восстанавливаешь файл из старого коммита:
git checkout HASH_КОММИТА -- file.txt

Фиксируешь это в текущем коммите:
git add file.txt
git commit -m "Restore file from previous commit"

git log --oneline --graph --all
